# Testing out open-source EDGAR parsers

I was hoping that parsing the older (pre-XBRL) SEC filings in EDGAR's archive would be something already done to a reliable manner by existing python packages.  At least from the main Google results I saw, none do this particularly well.  Existing parsers appear to mainly provide python structure to modern XBLR filings.


## Tried

None of these were up to the task:
- [edgar_tools](https://github.com/dgunning/edgartools)
- [sec_parser](https://github.com/alphanome-ai/sec-parser)
- [edgarParser](https://github.com/rsljr/edgarParser)
